# code review[^1] 
使用者：google/facebook/microsoft/alibaba/NetEase
## 意义
1. 知识共享
	1. 分享good smells, tricks，快速提高编码能力到同一水平
2. 规范
	1. 补充lint的不足：格式，可读性，可扩展，可维护，可复用性
3. 代码质量
	1. 逻辑错误，性能问题
4. 减缓工程腐坏速度
5. 摒弃个人主义
	1. 有哪些公用的方法？在哪些领域已经找到最佳解决方案？业务模型中有哪些隐藏的坑？
